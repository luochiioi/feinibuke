# P11 迭代提案 — 非遗内容富化与发现增强（2026-05-16）

> 本文是 **提案/方向草案**，不是实施计划。下个会话先用 brainstorming 技能与用户确认范围，再用 writing-plans 产出正式实施计划 `2026-05-XX-p11-*.md`。

## 背景

P10（非遗内容深化）已真机验收通过：打卡点升级为有内容的非遗条目，有详情页 / 名录页 / 后台编辑 / 10 条澳门湖南种子。
P10 明确**延后了视频**（范围边界决策：P10 只做图文，视频留 P11）。

当前 app 能力：地图打卡 → 任务/成就/路线/好友/排行 → 非遗详情/名录 → uni-admin 后台。

## P11 候选功能（待 brainstorm 确认取舍）

### F1 非遗视频内容（P10 明确延后项，建议纳入）
- 后台非遗编辑页支持上传视频；F1 详情页用 `<video>` 组件播放。
- 数据层：`tourism_heritage` 加 `videos: string[]`（或 `videoUrl`）字段。
- 媒体：复用 `photo-center.upload` folder 机制（`heritage-media/`）或评估视频大文件直传。
- 风险点：大文件 base64 上传可能超云函数限制 → 可能需 `uni.uploadFile` 直传云存储；UTS `<video>` 组件真机兼容性需先验证。

### F2 非遗名录搜索（轻量，建议纳入）
- 名录页 F2 顶部加搜索框，按非遗名称 / 传承人模糊搜索。
- 后端 `heritage-center.list` 加 `keyword` 参数（服务端 `db.RegExp` 或字段匹配）。
- 纯增量，无 schema 变更，风险低。

### F3 非遗主题路线（中等，可选）
- 把非遗打卡点编排成主题路线（如"澳门世遗探访""湘楚百工之旅"），复用现有 `routes` 系统。
- 路线详情页关联其经过的 heritage 条目，形成"路线 → 非遗点 → 详情"的浏览闭环。
- 涉及 routes 与 heritage 两个域的打通，工作量中等。

### 延后/备选（P11 暂不建议纳入）
- 传承人信息补录（P10 种子未填传承人，避免杜撰；可后台逐步补）。
- 非遗收藏 / 点赞 / 分享。
- 多语言（澳门项目有葡语受众潜在需求）。

## 建议的 P11 主线

**F1 非遗视频 + F2 名录搜索** 为 P11 核心（一个延后项 + 一个轻量发现增强，范围可控）；F3 主题路线视精力决定是否纳入或留 P12。

下个会话 brainstorm 时需与用户确认的关键决策：
1. 视频走 `photo-center` base64 上传还是 `uni.uploadFile` 直传？（取决于视频体积上限）
2. 视频字段：`tourism_heritage` 加 `videos: string[]` 还是单 `videoUrl`？
3. F3 主题路线是否纳入 P11？
4. 搜索是否同时支持后台非遗列表页（admin）？
